3. Add Docker Hub Credentials

Since the pipeline needs to push images to Docker Hub, you must store your token securely.

    From the Jenkins Dashboard, go to Manage Jenkins -> Credentials.

    Click on (global) under the Domains column.

    Click Add Credentials (top right).

    Kind: Username with password.

    Username: 4kanksh

    Password: [Paste your Docker Hub Personal Access Token].

    ID: dockerhub-credentials (This must match the name in the Jenkinsfile provided earlier).

    Click Create.

4. Create the Pipeline Job

    Click New Item on the dashboard.

    Enter name: docker-python-pipeline.

    Select Pipeline and click OK.

    Scroll down to the Pipeline section.

    Definition: Pipeline script.

    Paste the following optimized code into the script box:

Groovy

pipeline {
    agent any
    environment {
        DOCKER_IMAGE = '4kanksh/my-python-app'
    }
    stages {
        stage('Clone') {
            steps {
                git branch: 'master', url: 'https://github.com/4kanksh/lab5-pp.git'
            }
        }
        stage('Build') {
            steps {
                sh "docker build -t $DOCKER_IMAGE:latest ."
            }
        }
        stage('Push') {
            steps {
                withCredentials([usernamePassword(credentialsId: 'dockerhub-credentials', usernameVariable: 'USER', passwordVariable: 'PASS')]) {
                    sh "echo $PASS | docker login -u $USER --password-stdin"
                    sh "docker push $DOCKER_IMAGE:latest"
                }
            }
        }
        stage('Deploy') {
            steps {
                sh 'docker stop myapp-container || true'
                sh 'docker rm myapp-container || true'
                sh "docker run -d -p 5000:5000 --name myapp-container $DOCKER_IMAGE:latest"
            }
        }
    }
}

1. The Port Conflict Error

In your manual steps, you run:
docker run -d -p 5000:5000 --name container ...

In your Jenkins Pipeline, you also run:
sh "docker run -d -p 5000:5000 --name myapp-container ..."

The Error: Jenkins will fail during the Deploy stage because Port 5000 is already being used by your manual container.

The Fix: Before running the Jenkins build, stop and remove any existing containers using that port:
Bash

sudo docker stop container
sudo docker rm container

2. Docker Socket Permission Error

Even though you are running Jenkins as your user, the Jenkins process often lacks direct permission to talk to the Docker engine.

The Error: docker: permission denied or cannot connect to the Docker daemon.

The Fix: Run this command in your terminal once:
Bash

sudo chmod 666 /var/run/docker.sock
